objective: |
  【課題】scripts/approve.py に HITL Gate 1/2/3 の実際の承認関数が存在しない。
  tests/test_hitl.py が `from scripts.approve import approve_gate_1, approve_gate_2, approve_gate_3` で ImportError を起こし、テストスイート全体がブロックされている。
  さらに、現状の approve.py は log_approval() と main() のみで、承認しても JOB-###.md の YAML frontmatter の `status` を更新しないため、Gate 承認が形式的に機能しない。

  【実装要件】
  1. YAML frontmatter の読み書きヘルパー `_read_frontmatter(job_path: Path) -> tuple[dict, str]` と `_write_frontmatter(job_path: Path, fm: dict, body: str) -> None` を追加する。
     - 読み込み: `---\nkey: value\n---\nbody` を split し、yaml.safe_load で frontmatter dict を取得。存在しなければ空 dict + 全文 body を返す。
     - 書き込み: `f"---\n{yaml.dump(fm, allow_unicode=True)}---\n{body}"` を組み立て、utils.atomic_io.atomic_write() で原子的に書き込む。
  2. `approve_gate_1(job_path: Path, approver: str) -> None` を実装。
     - frontmatter の `status` を `"approved_gate_1"` に更新。
     - `approved_by` に approver を記録。
     - `log_approval(job_id, 1, True)` を呼び出す。
  3. `approve_gate_2(job_path: Path, approver: str, reject: bool = False) -> None` を実装。
     - reject=False 時: `status` を `"staged"` に更新、`approved_gate_2_by` に approver を記録。
     - reject=True 時: `status` を `"gate_2_rejected"` に更新、`gate_2_rejected_by` に approver を記録。
     - `log_approval(job_id, 2, not reject)` を呼び出す。
  4. `approve_gate_3(job_path: Path, approver: str) -> None` を実装。
     - frontmatter の `status` を `"promoted"` に更新。
     - `approved_gate_3_by` に approver を記録。
     - `log_approval(job_id, 3, True)` を呼び出す。
  5. `tests/test_hitl.py` を更新し、import エラーが解消され、各 Gate 関数が frontmatter の status を正しく更新することを検証する。
     - テストは `pytest tests/test_hitl.py -v` で全て通過すること。
  6. `git commit` を実行し、作業ディレクトリをクリーンに保つこと。

  【制約】
  - すべてのファイル書き込みは `utils.atomic_io.atomic_write()` を使用（tempfile + fsync + shutil.move）。
  - Python 3.12 必須。
  - 既存の `log_approval()` 関数と `main()` CLI の動作は壊さないこと。
  - 新規 import は `yaml`（PyYAML）のみ追加可。
