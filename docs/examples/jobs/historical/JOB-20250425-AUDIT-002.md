objective: |
  【課題】制御面の失敗が成功扱いになる構造欠陥が複数存在する。
  1. graph.py の audit() は audit_result="FAIL" でも status="done" を返し、監査失敗ジョブが完了扱いになって昇格される可能性がある。
  2. wiki_daemon.py の rebuild_state() は起動時に work/jobs/*.md を全て "queued" に再設定し、done/failed などの終端状態ジョブが再実行される。
  3. wiki_daemon.py の is_lock_stale() は PID が生きていても 10分超で stale と判定し、長時間 CrewAI ジョブのロックを奪って二重実行を引き起こす。

  【実装要件】
  1. graph.py の audit() ノード修正:
     - issues が空でない場合: return {..., "audit_result": "FAIL: ...", "status": "audit_failed", "error": "FAIL: ..."}
     - issues が空の場合: return {..., "audit_result": "PASS", "status": "done", "error": None}
     - "audit_failed" を新たな終端状態として追加する。

  2. wiki_daemon.py の終端状態統一:
     - モジュールレベル定数 `TERMINAL_STATUSES = {"done", "failed", "audit_failed", "promoted"}` を追加。
     - process_jobs() のスキップ条件を `if job_info.get("status") in TERMINAL_STATUSES:` に変更。

  3. wiki_daemon.py の daemon 状態管理修正:
     - `reconcile_state(state: dict) -> dict` 関数を新規実装。
       - 既存 state["jobs"] からファイルが消えたジョブを削除。
       - work/jobs/*.md を走査し、job_id が state["jobs"] に存在しない新規ファイルのみ status="queued" で追加。
       - 既存ジョブの status は終端状態であっても上書きしない。
     - `load_state()` を修正: STATE_FILE が存在すれば `json.load` 後に `reconcile_state(state)` を呼び出して返す。存在しなければ `reconcile_state({"jobs": {}})` を返す。
     - `main()` の `state = rebuild_state()` を `state = load_state()` に変更。
     - `rebuild_state()` 関数自体は「新規ファイルを queued として追加する補助」として残し、`reconcile_state()` 内で利用してもよいが、`main()` からの直接呼び出しは消す。

  4. wiki_daemon.py の stale-lock 安全性修正:
     - `is_lock_stale()` の判定ロジックを変更:
       - `if pid_alive: return False`（生きているプロセスのロックは絶対に奪わない）
       - `return age_min > STALE_MINUTES`（死んでいるプロセスのロックのみ年齢で判定）
     - `reclaim_stale_lock()` に `is_lock_stale()` の再確認を追加: `if not is_lock_stale(job_id): return False` を先頭に入れ、二重チェックとする。

  5. テスト更新:
     - tests/test_domain_isolation.py または新規 tests/test_daemon_safety.py で以下を検証:
       - audit_failed 状態が graph.py から正しく返されること。
       - TERMINAL_STATUSES に含まれるジョブが process_jobs() でスキップされること。
       - 生存中 PID のロックが is_lock_stale() で False になること。
       - reconcile_state() が既存 done/failed ジョブの status を "queued" に上書きしないこと。
     - `pytest tests/ -v` で全テスト通過を確認すること。
