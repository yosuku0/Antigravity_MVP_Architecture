objective: |
  【課題】セキュリティ境界とリソース管理に3つの構造的欠陥がある。
  1. graph.py の squad_router() が domain 未指定、.domain 読み込み失敗、例外発生時に全 squad 実行へフォールバックしており、Domain 境界が fail-open になっている（個人情報ドメインで全 squad が実行される可能性）。
  2. browser_tool.py で browser_use.Browser() が生成された後に browser.close() が呼ばれず、長時間稼働 daemon で Chromium プロセスがリークし続けてメモリ枯渇・OOM が発生する。
  3. scripts/ingest.py が subprocess.run(["npx.cmd", "lit", "parse", ...]) を使用しており、scripts/scope_guard.py の FORBIDDEN 辞書が "subprocess": "Use e2b sandbox for execution" と定義しているため、システム自身が禁止ルールに違反している自己矛盾状態にある。

  【実装要件】
  1. graph.py の squad_router() を fail-closed に修正:
     - KnowledgeOS() の初期化・.domain 読み込み・meta 解決を try/except で囲む。
     - `domain` が未指定（None/空文字）の場合: return {..., "status": "failed", "error": "Domain not specified in job frontmatter"}
     - `meta` が None（.domain ファイルなしまたは不明ドメイン）の場合: return {..., "status": "failed", "error": f"Unknown domain or missing domain metadata: {domain}"}
     - `meta.allowed_squads` が空または許可 squad と有効 squad の積集合が空の場合: return {..., "status": "failed", "error": f"No valid squads allowed for domain: {domain}"}
     - 例外発生時（KnowledgeOS 初期化失敗等）: return {..., "status": "failed", "error": f"Domain routing failed for {domain}: {e}"}
     - フォールバックでの「全 squad 実行」パスを完全に削除する。

  2. apps/crew/squads/research_squad/tools/browser_tool.py のブラウザリーク修復:
     - WebResearchTool._run()（または _arun_research）で Browser() 生成後、try/finally で browser.close() を確実に呼び出す。
     - コードパターン:
       ```python
       browser = Browser()
       try:
           agent = Agent(task=task, llm=llm, browser=browser)
           result = await agent.run()
           return str(result)
       finally:
           await browser.close()
       ```
     - browser.close() が awaitable か同期メソッドかを確認し、存在しない場合は `browser.close()` または `browser.stop()` の呼び出しを確認して最も安全な方法を使用する。

  3. scripts/ingest.py の subprocess 自己矛盾解消:
     - `subprocess.run(["npx.cmd", "lit", "parse", ...])` の呼び出しを削除する。
     - vendor/liteparse/ にクローン済みの liteparse ライブラリを使用して同等のドキュメント解析処理に置き換える。
     - liteparse のインポートパスと使用方法は vendor/liteparse/ の README またはソース構造を確認して実装する。
     - liteparse が使用不可の場合は、最小限の Python 標準ライブラリ（pathlib, re 等）による frontmatter/header 抽出にフォールバックし、それでも subprocess は使用しない。
     - scope_guard.py の FORBIDDEN チェックが ingest.py に対して「subprocess 使用」で検出されないことを確認する。

  4. テスト・動作確認:
     - graph.py の squad_router() fail-closed 化を検証するテストを tests/ に追加または既存テストを更新:
       - domain 未指定ジョブが status="failed" になること。
       - 存在しないドメイン指定が status="failed" になること。
       - .domain 読み込み失敗時に全 squad が実行されないこと。
     - pytest tests/ -v で全テスト通過を確認。
     - browser_tool.py の変更については、単体での import エラーがないことと、Browser クラスの close() メソッド呼び出し構文が正しいことを確認。
     - ingest.py の変更については、`python scripts/ingest.py --help` または最小入力で動作することを確認（subprocess import が削除されていることも確認）。

  5. git commit:
     - 修正完了後 `git commit -m "fix(security): domain fail-closed routing, browser leak cleanup, remove subprocess self-contradiction"` を実行。
